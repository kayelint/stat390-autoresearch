# Manga → Anime Adaptation Classifier

- Predicts whether a manga will be adapted into an anime using an AutoResearch agent loop.
- Agent iteratively improves an sklearn classification pipeline with a keep/discard workflow.

---

## Problem

- **Task:** Binary classification — will this manga get adapted into an anime?
- **Metric:** Validation ROC-AUC (primary), F1 Score (secondary)
- **Data:** ~400 manga pulled from the MyAnimeList API

**Important — no target leakage:** Features are restricted to information known
*before* adaptation (genre, magazine, chapter count, start year, media type).
Score, rank, popularity, and member counts are excluded as these are
consequences of adaptation, not predictors.

---

## Project Structure

```
autoresearch/
├── prepare.py                      # FROZEN — data loading, metric, logging, plotting
├── model.py                        # EDITABLE — agent modifies only this file
├── run.py                          # Run one experiment and log result
├── program.md                      # Agent instructions
├── results.tsv                     # Experiment log (auto-generated)
├── performance.png                 # Performance plot (auto-generated)
└── data_final.ipynb                # Dataset (generated by data notebook)
```

**Key rule:** The agent may only modify `model.py`. Everything else is frozen.

## Data Split

| Split | Size | Purpose |
|-------|------|---------|
| Train | 70%  | Model fitting |
| Val   | 15%  | Keep/discard decisions during agent loop |
| Test  | 15%  | **LOCKED** — final numbers only, reported once at the end |

The test set is stratified and fixed by `RANDOM_SEED = 42` in `prepare.py`.
Never evaluate on the test set during the agent loop.

## How to Run the Agent Loop

### Install Claude Code (recommended)

```bash
# macOS / Linux / WSL
curl -fsSL https://claude.ai/install.sh | bash

# Then launch from this directory
cd manga_autoresearch
claude
```

### Prompt to paste into your agent

```
Read program.md for your instructions, then read model.py.
Run `python run.py "baseline: RandomForest" --baseline` to establish the baseline AUC.

Then enter the AutoResearch loop:
1. Propose one modification to model.py
2. Edit model.py with your change
3. Run: python run.py "<short description of what you changed>"
4. Compare val_auc to current best
   - If improved: KEEP the change, note the new best
   - If worse: REVERT model.py to the previous version
5. Repeat from step 1. Try at least 8 different ideas.

After all iterations, run `python prepare.py` to generate performance.png.
Print a summary table of all experiments with keep/discard decisions.
DO NOT call evaluate() on the test set (X_test) during the loop.
```

## Features Used

| Feature | Type | Notes |
|---------|------|-------|
| `num_volumes` | numeric | Total volumes |
| `num_chapters` | numeric | Total chapters |
| `start_year` | numeric | Year serialization began |
| `run_years` | numeric | Duration of serialization |
| `in_jump_magazine` | binary | Published in any Jump title |
| `in_major_magazine` | binary | Published in major magazine |
| `genre_*` (×18) | binary | One flag per genre |
| `media_type` | categorical | manga / manhwa / novel / one_shot |
| `status` | categorical | finished / currently_publishing |
| `nsfw` | categorical | white / gray / black |

**Intentionally excluded (target leakage):**
`score`, `rank`, `popularity`, `num_list_users`, `num_scoring_users`, `num_favorites`
